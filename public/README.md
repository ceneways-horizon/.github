### Product Portfolio

**[RAG Axis](https://github.com/ceneways-horizon/rag-axis)**

- License: Apache 2.0

- Status: In Development (v1.0 in progress)
- Language: Python
- License: Apache 2.0

## Our Standard

We hold every product to the same production standards:

### Reliability First
Systems fail loudly, not silently. Issues are caught before they become costly.

### Built for Real Organizations
Products are designed around the constraints real organizations operate under: compliance, scale, existing systems.

### Long-Term Thinking
We design for the next decade of technological change, not the current AI hype cycle.

### Observable by Design
Cost tracking, audit trails, decision logs—systems are transparent and reproducible.

### Typed and Testable
Type safety and comprehensive testing are non-negotiable.

### Contributing

We welcome contributions to our open-source projects. See [CONTRIBUTING.md](./CONTRIBUTING.md) for guidelines.

**Before contributing:**
- Read our [Code of Conduct](./CODE_OF_CONDUCT.md)
- Check [open issues](https://github.com/ceneways-horizon/rag-axis/issues) for what needs work
- Follow our [development standards](#development-standards)

### Development Standards

**Code Quality:**
- Type hints required (Python: mypy strict)
- Tests required (90%+ coverage)
- Docstrings for all public APIs
- Formatted with black + isort

**Commits:**
- Semantic commit messages
- Atomic commits (one feature per commit)
- Link to issues when applicable

**Pull Requests:**
- Describe what changed and why
- Reference related issues
- One PR = one feature or fix
- Approval required before merge
